# 2021a SSH-mapping OSE — Methods & Data Inventory

Recon to support choosing which gridding method to implement in `sverdrup`.
Generated by driving the challenge's own eval (`their_eval.score`) over every
reconstruction map reachable from the surviving MEOM mirror.

## Data access reality (2026-06-27)

- **ODC THREDDS** (`tds.aviso.altimetry.fr`) — **dead** (host unresolvable
  globally, on every machine tried). Was the original challenge data home.
- **MEOM mirror** (`ige-meom-opendap.univ-grenoble-alpes.fr/.../2021a-SSH-mapping-OSE`)
  — **alive, unauthenticated.** Carries `dc_obs/` (all 7 along-track L3 files)
  and `dc_maps/` (7 reconstruction maps). **Does NOT carry BASELINE or DYMOST.**
- **AVISO SFTP** (`sftp://ftp-access.aviso.altimetry.fr:2221`) — **alive**
  (AVISO creds), but holds AVISO **operational/experimental** products only
  (DUACS grids, along-track, eddy trajectories, …), **not** the ODC
  data-challenge reconstruction maps.

Net: the OI **BASELINE** map and the **DYMOST** map are unobtainable from any
reachable source. This does not block the milestone — `sverdrup`'s own OI run
*produces* the BASELINE-equivalent map (that is the deliverable).

## Downloaded locally (`data/2021a_ssh_mapping_ose/`, git-ignored, ~1.0 GB)

- `dc_obs/` (116 MB, 7 files): 5 mapping missions — `alg` (SARAL/AltiKa),
  `j2g`+`j2n` (Jason-2 geodetic + interleaved), `j3` (Jason-3), `s3a`
  (Sentinel-3A), `h2g` (Haiyang-2A) — plus the withheld eval track `c2`
  (Cryosat-2). All span 2016-12-01 → 2018-01-31 over the 285-315/23-53 box.
- `dc_maps/` (925 MB, 7 files): `DUACS`, `MIOST`, `BFN`, `4dvarNet_2022`,
  `neurost_ssh-sst`, `convlstm_ssh`, `convlstm_ssh-sst`. (BFN is 700 MB.)

## Methods, published leaderboard, and reproduced scores

Scores are µ(RMSE) / σ(RMSE) / λx(km); higher µ + lower λx = better. "Reproduced"
= our `their_eval.score(map, c2_track)` on the shipped map (eval region
295-305/33-43, 2017). Eval pipeline runs on modern pyinterp via faithful
API-compat shims (see `parameter_audit_trail.md`).

**Eval validation:** the three classical methods whose maps survive (DUACS,
MIOST, BFN) all reproduce their published rows within tolerance — independent
3× confirmation that our eval harness == the challenge's ground-truth eval.

## Implementation-choice notes

- **OI / BASELINE** is already the `sverdrup` milestone target — it is the
  natural first reference; we don't need their map, we produce it.
- **DUACS** is the same OI family with operational DT2018 covariances; the
  cheapest "working reference" upgrade from BASELINE, map on disk to validate.
- **MIOST** (multiscale OI) is the closest *algorithmic* extension of our OI
  that buys resolution; map on disk.
- **BFN / DYMOST** are dynamical (QG) — a different engine from OI; BFN map on
  disk (DYMOST map unavailable).
- **ML methods** (neurost/convlstm/4dvarNet) score best but are a separate
  stack; maps on disk as targets, but out of scope for the OI engine.
